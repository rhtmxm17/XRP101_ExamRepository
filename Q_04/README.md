# 4번 문제

주어진 프로젝트는 다음의 기능을 구현하고자 생성한 프로젝트이다.

## 기능

### 1. Player

- 상태 패턴을 사용해 Idle 상태와 Attack 상태를 관리한다.
- Idle상태에서 Q를 누르면 Attack 상태로 진입한다
  - 진입 시 2초 이후 지정된 구형 범위 내에 있는 데미지를 입을 수 있는 적을 탐색해 데미지를 부여하고 Idle상태로 돌아온다
- 상태 머신 : 각 상태들을 관리하는 객체이며, 가장 첫번째로 입력받은 상태를 기본 상태로 설정한다.

### 2. NormalMonster

- 데미지를 입을 수 있는 몬스터

### 3. ShieldeMonster

- 데미지를 입지 않는 몬스터

위 기능들을 구현하고자 할 때
제시된 프로젝트에서 발생하는 `문제들을 모두 서술`하고 올바르게 동작하도록 `소스코드를 개선`하시오.

## 답안

### Player

`Physics.Overlap`의 대상에 자기 자신이 포함되어있다. 문제에서 범위 내의 _유닛_ 이 아닌 _적_ 을 탐색한다고 되어있기 때문에 이는 의도하지 않은 동작이다.  
충돌체 감지시 `IDamagable` 인터페이스를 가지고 있는지 여부와 무관하게 `TakeHit()`을 호출하고있다.
